# Тест-план

## 1. Идентификатор тест-плана

`TP-EMAIL-AI-2026-01` — тест-план преддипломной практики, весна 2026.

## 2. Введение

Тест-план покрывает автоматизированные (pytest) и ручные (GUI) проверки
AI-приложения для анализа почты. Основан на структуре из методички
МУИВ (16 разделов) и ISO/IEC/IEEE 829.

## 3. Объект тестирования

- Python-пакет `email_analyzer` (ML, mail, db, backend).
- REST API на FastAPI.
- PyQt6-клиент.

## 4. Функции, которые будут протестированы

- Классификатор писем (TF-IDF + LogReg).
- Детектор спама и фишинга.
- Суммаризатор.
- Извлечение сущностей.
- Парсер MIME.
- IMAP-клиент (через FakeBackend).
- Оркестрирующий сервис анализа.
- Аутентификация (bcrypt + JWT).
- Экспорт отчётов (.docx, .xlsx).
- REST API (register/login/analyze/admin).
- Smoke-тесты клиента (все окна импортируются).

## 5. Функции, которые не будут протестированы автоматически

- Реальный сетевой IMAP (зависит от внешнего сервера).
- Визуальный рендер PyQt6 (нет дисплея в CI — только smoke-тесты импорта).
- Сборка `.exe` через PyInstaller (проверяется ручным запуском).

## 6. Тестовые подходы

- Модульные тесты для всех компонентов ML и db.
- Интеграционные тесты REST API через `TestClient` + in-memory SQLite.
- Smoke-тесты импорта клиентских окон.
- Ручное UI-тестирование по чек-листу (см. раздел 13).

## 7. Критерии прохождения / непрохождения

- **Pass:** все тесты pytest — зелёные, покрытие ≥ 70% по `src/email_analyzer`.
- **Fail:** хотя бы один тест красный или падение при запуске клиента.

## 8. Критерии приостановления и возобновления

- Приостановка: критический баг в ядре (классификатор не обучается).
- Возобновление: после фикса и прогона всех тестов.

## 9. Результаты тестирования

- Отчёт pytest (`pytest --tb=short`).
- Отчёт покрытия (`pytest --cov`).
- Снимки экрана GUI-тестирования (для отчёта ПДП).

## 10. Задачи тестирования

1. Подготовить тестовые данные (seed-набор писем).
2. Прогнать автотесты.
3. Вручную протестировать UI по чек-листу.
4. Зафиксировать результаты в отчёте.

## 11. Окружение

- Python 3.11+ (рекомендовано; реально работает на 3.14 с предупреждениями).
- Виртуальное окружение `.venv`.
- SQLite — встроено в Python.
- PostgreSQL 15+ — только для интеграционного теста prod-конфигурации.

## 12. Обязанности

- Разработчик (автор): написание и прогон автотестов, ручное UI-тестирование.
- Руководитель практики: приёмка.

## 16. Тест-кейсы

### TC-01 Классификатор: рабочее письмо

- **PreConditions:** `EmailClassifier.fit_seed()` выполнен.
- **Steps:**
  1. Вызвать `.predict("Прошу согласовать смету по проекту")`.
- **Expected result:** `result.category ∈ {WORK, IMPORTANT}`, `0 ≤ confidence ≤ 1`.

### TC-02 Спам-детектор: обычное письмо

- **PreConditions:** `SpamDetector()` создан.
- **Steps:** `detect("Встреча в 10:00", "Приходи в офис")`.
- **Expected result:** `is_spam = False`, `score < 0.5`.

### TC-03 Спам-детектор: явный спам

- **PreConditions:** `SpamDetector()` создан.
- **Steps:** `detect("ВЫИГРАЛИ МИЛЛИОН!!!", "Промокод SPAM")`.
- **Expected result:** `is_spam = True`, `reasons` не пусты.

### TC-04 Фишинг с поддельным отправителем

- **PreConditions:** `SpamDetector()` создан.
- **Steps:** `detect("…", "…", sender="sec@paypal-fake.ru")`.
- **Expected result:** `is_phishing = True`.

### TC-05 Саммаризация

- **PreConditions:** `Summarizer(max_sentences=2)`.
- **Steps:** summarize(5 предложений).
- **Expected result:** не больше 2 предложений в выдаче.

### TC-06 Извлечение даты/суммы

- **PreConditions:** `EntityExtractor()`.
- **Steps:** extract("Оплатите 1 500 рублей до 01.05.2026").
- **Expected result:** в `amounts` есть сумма, в `dates` — дата.

### TC-07 Парсер MIME

- **PreConditions:** —
- **Steps:** `parse_email_bytes(raw)` с валидным письмом.
- **Expected result:** заполнены все поля ParsedEmail.

### TC-08 IMAP-клиент с FakeBackend

- **PreConditions:** `FakeBackend([3 письма])`.
- **Steps:** `fetch_recent(limit=10)`.
- **Expected result:** вернулось 3 ParsedEmail.

### TC-09 Регистрация + логин через API

- **PreConditions:** in-memory SQLite, FastAPI TestClient.
- **Steps:**
  1. POST /auth/register {alice}.
  2. POST /auth/login {alice}.
  3. GET /auth/me.
- **Expected result:** все 3 — 200; `/me.username == "alice"`.

### TC-10 Неверный пароль — 401

- **PreConditions:** alice уже зарегистрирована.
- **Steps:** POST /auth/login с неверным паролем.
- **Expected result:** 401 Unauthorized.

### TC-11 `/analyze/email` без токена — 401

- **PreConditions:** TestClient.
- **Steps:** POST /analyze/email без Authorization.
- **Expected result:** 401.

### TC-12 Админская ручка закрыта для не-admin

- **PreConditions:** user без роли admin, валидный токен.
- **Steps:** GET /admin/users.
- **Expected result:** 403 Forbidden.

### TC-13 Админская ручка открыта для admin

- **PreConditions:** admin-пользователь.
- **Steps:** GET /admin/users.
- **Expected result:** 200, в списке видны все пользователи.

## 17. Баг-репорт (степени критичности)

- **S1 — Blocker.** Приложение не запускается.
- **S2 — Critical.** Часть функционала не работает (импорт падает).
- **S3 — Major.** Некорректная классификация для значимого класса писем.
- **S4 — Minor.** Визуальные огрехи, не влияющие на функционал.
- **S5 — Trivial.** Опечатки в подсказках.
